# Support Policy

FoggyKitchen public repositories are provided as **best-effort, open source reference implementations**.

They are intended for:

- learning
- evaluation
- reuse at your own discretion
- adaptation into your own delivery patterns

They are **not** delivered with a guaranteed support response time, SLA, or production-readiness warranty.

---

## What You Can Expect

- You can open GitHub Issues for bugs, questions, or documentation gaps.
- Pull requests are welcome.
- Maintainer responses are handled on a **best-effort** basis.
- Some issues may be answered quickly, some later, and some may remain unresolved.

---

## No SLA or Guaranteed Support

Unless you have a separate paid agreement with FoggyKitchen:

- there is **no SLA**
- there is **no guaranteed response time**
- there is **no guaranteed fix delivery**
- there is **no guarantee that an example or pattern will work unchanged in your environment**

Each environment differs in:

- tenancy configuration
- quotas
- IAM policies
- provider versions
- region availability
- organizational security constraints

You are responsible for validating the examples before production use.

---

## How to Open a Useful Issue

If you open a GitHub Issue, include:

- repository name and branch or tag
- Terraform / OpenTofu version
- provider version
- affected example or pattern
- sanitized payload or variable inputs
- exact error output
- reproduction steps

This substantially improves the chance of a useful response.
